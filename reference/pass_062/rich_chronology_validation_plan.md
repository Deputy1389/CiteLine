# Rich Chronology Validation Plan

## Purpose
Prove the improved segmentation actually preserves attorney-useful treatment progression on richer packets.

## Minimum packet classes
- ED -> imaging -> follow-up
- PT course packet
- injection or surgery packet
- mixed prior-history packet

## Acceptance
- clinically distinct phases remain separate
- chronology is not collapsed into summary-only nodes
- no return of junk front-page anchors
- review gates remain stable

## Output
- per-packet chronology row counts
- before/after event count comparisons where available
- PDF spot checks for treatment progression
